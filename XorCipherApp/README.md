# Проект XOR-шифрования текста (C# Windows Forms)

## Описание
Программа для кодирования и декодирования текста с использованием XOR-шифрования с современной архитектурой и адаптивным интерфейсом в оттенках серого.

## Функциональность
- **Ручной ввод текста**: Возможность ввода шифруемого сообщения непосредственно в окне программы
- **Загрузка из файла**: Загрузка исходного текста из внешнего файла (.txt)
- **Шифрование**: Кодирование текста с использованием XOR-алгоритма и пользовательского ключа
- **Дешифрование**: Расшифровка зашифрованного текста (в HEX-формате) с тем же ключом
- **Отображение результатов**: Вывод результатов шифрования/дешифрования в окне программы
- **Сохранение в файл**: Сохранение результатов работы в отдельный файл
- **Адаптивный интерфейс**: Автоматическая подстройка под размер окна
- **Улучшенные скроллы**: Визуально улучшенные полосы прокрутки в текстовых полях

## Требования
- .NET 8.0 SDK
- Windows (для запуска Windows Forms приложения)

## Архитектура проекта
Проект использует расширяемую архитектуру с разделением на слои:

```
XorCipherApp/
├── Core/                    # Ядро - бизнес-логика
│   └── IXorCipher.cs        # Интерфейс и реализация XOR-шифрования
├── Models/                  # Модели данных
│   └── CipherResult.cs      # Модели результатов и конфигурации
├── Services/                # Сервисы - внешние операции
│   └── IFileService.cs      # Интерфейс и реализация работы с файлами
├── UI/                      # Пользовательский интерфейс
│   ├── ColorPalette.cs      # Цветовая палитра (оттенки серого)
│   ├── ControlFactory.cs    # Фабрика элементов управления
│   └── ResponsiveForm.cs    # Базовый класс адаптивной формы
├── Program.cs               # Точка входа приложения
├── MainForm.cs              # Основная форма приложения
└── XorCipherApp.csproj      # Файл проекта
```

### Преимущества архитектуры
- **Dependency Injection**: Легко заменять реализации (например, для тестирования)
- **Single Responsibility**: Каждый класс отвечает за одну задачу
- **Extensibility**: Простое добавление новых функций без изменения существующего кода
- **Testability**: Бизнес-логика изолирована от UI

## Как использовать

### Сборка проекта
```bash
dotnet build
```

### Запуск
```bash
dotnet run
```

### Работа с программой

1. **Шифрование текста:**
   - Введите текст в поле "Исходный текст" или загрузите из файла
   - Укажите ключ шифрования (по умолчанию: "SecretKey123")
   - Нажмите кнопку "Зашифровать"
   - Результат отобразится в формате HEX в поле "Результат"

2. **Дешифрование текста:**
   - Вставьте зашифрованный текст (HEX-формат) в поле "Зашифрованный текст"
   - Укажите тот же ключ, который использовался при шифровании
   - Нажмите кнопку "Расшифровать"
   - Расшифрованный текст отобразится в поле "Результат"

3. **Сохранение результатов:**
   - Нажмите кнопку "Сохранить результат в файл"
   - Выберите место сохранения и имя файла

4. **Очистка:**
   - Нажмите кнопку "Очистить всё" для сброса всех полей

## Примечания
- Локальная база данных **не требуется** - все операции выполняются с файлами напрямую
- Результаты шифрования отображаются в HEX-формате для корректного отображения бинарных данных
- Для успешного дешифрования необходимо использовать тот же ключ, что и при шифровании
- Интерфейс автоматически адаптируется под размер окна
- Используется современная цветовая схема в оттенках серого вместо фиолетового

## Алгоритм XOR
XOR (исключающее ИЛИ) - симметричный алгоритм шифрования, где:
- Шифрование: `зашифрованный_байт = исходный_байт XOR ключ_байт`
- Дешифрование: `исходный_байт = зашифрованный_байт XOR ключ_байт`

Ключ циклически повторяется для шифрования текста любой длины.

## Расширение функциональности

### Добавление нового алгоритма шифрования
1. Создайте новый класс в папке `Core/`, реализующий интерфейс `IXorCipher`
2. Передайте его в конструктор `MainForm` при создании приложения

### Добавление нового сервиса
1. Создайте интерфейс в папке `Services/`
2. Реализуйте его в том же файле или отдельном классе
3. Используйте через dependency injection в `MainForm`

### Изменение цветовой схемы
Отредактируйте статические цвета в классе `ColorPalette` в файле `UI/ColorPalette.cs`
